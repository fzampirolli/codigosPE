Paradigma Estruturado¶

  • Uns dos primeiros estilos (ou paradigma) para estruturar um Software chamado de Programao Estruturada.

  • Neste paradigma estruturado, o programador define Tabelas (ou Registros ou tambm chamadas Entidades), onde se podem armazenar variveis de vrios tipos de dados,

    • diferentemente dos vetores e matrizes vistos nos captulos anterios, onde todos os dados devem ser de um mesmo tipo de dado (exceto as listas em Python).

  • Por exemplo, possvel definir uma tabela para armazenar as informaes de um aluno num contexto de um sistema acadmico, chamada Aluno.

  • Esta tabela Aluno, na verdade, pode ser considera um novo tipo de dados e possvel, por exemplo, instanciar uma varivel (ou registro), como Julia do tipo Aluno.

  • Como atributos da tabela Aluno, possvel ter nome, matrcula, data de nascimento, ano de ingresso, etc.

  • Observe que na tabela Aluno importante tambm ter informao de curso e de disciplinas j cursadas.

  • Estas informaes podem ser instncias de outras tabelas, como Curso e Disciplina, com seus respectivos atributos apropriados.

  • Em Banco de Dados estas instncias so possveis atravs de um atributo representando a chave (estrangeira) de outra tabela.

  • Alm disso, cada tabela possui um atributo (chamado chave primria) que identifica o registro, por exemplo, Julia, que pode estar armazenada num registro de nmero 33.

  • Resumindo, as estrutura definidas pelo programador podem armazenar em tempo de execuo as tabelas definidas em Bancos de Dados (com informaes armazenadas em disco).

Diagrama Entidade Relacionamento¶

  • Para poder visualizar melhor esta relao entre as tabelas Aluno, Curso e Disciplina, existe um modelo apropriado, chamado Diagrama Entidade Relacionamento (DER), como apresentado na figura abaixo.

  • Observe o relacionamento Aluno faz Disciplina(s).

  • Se for considerar um contexto onde um aluno est cursando uma disciplina, existe a necessidade de incluir uma nova tabela, chamada, por exemplo, Turma.

  • Assim, Turma possui Aluno(s) e Disciplina possui Turma(s).

  • Desse modo, para um sistema acadmico completo, necessrio criar vrias tabelas e seus relacionamentos.

  • Todos esses dados das tabelas devem ser lidos (digitados) uma vez e armazenados em disco.

  • Seno, toda vez que esse sistema acadmico for executado, todos os dados devero ser lidos novamente.

  • Existem vrias formas de guardar os dados de variveis em disco:

    • arquivo texto,
    • arquivo binrio ou
    • tabelas em um Sistema de Gerenciamento de Banco de Dados (SGBD), como Oracle, JDBC, SQL, PostgreSQL, etc.

  • Nas disciplinas de Banco de Dados, o aluno aprende como criar essas tabelas em um SGBD e tambm como us-las dentro de uma linguagem de programao.

  • Essa forma de organizar vrios tipos de dados em tabelas chama-se encapsulamento de dados.

  • O grande desafio dos programadores e analistas de sistemas definir corretamente essas tabelas e os seus relacionamentos. Os sistemas bem modelados so mais fceis de realizar manutenes.

  • Alm disso, os programadores devem criar estruturas, funes e procedimentos de forma bem organizada, por exemplo, usando vrios arquivos organizados em pastas.

  • O paradigma estruturado no facilita completamente esta organizao. Isso j no ocorre no paradigma orientado a objetos, resumido a seguir.

Paradigma Orientado a Objetos¶

  • Na programao estruturada no existe uma forma eficiente de organizar (encapsular) as funes especficas de uma tabela, como ocorre no encapsulamento dos dados, visto na programao estruturada.

  • Assim, surgiu a necessidade de criar um novo paradigma de programao, que a Programao Orientada a Objetos (POO), onde, alm de encapsular os dados, possvel encapsular as funes ou mtodos que os processam.

  • Na POO, tabela passou a se chamar Classe, e um registro de uma tabela passou a ser chamado de instncia de uma classe, tambm chamado de objeto.

  • Como no exemplo anterior, para o sistema acadmico, possvel criar a classe Aluno (que um novo tipo de dados) e com ela instanciar uma varivel Julia da classe Aluno.

Exemplo 01 - Criar um registro de Aluno¶

Exemplo para criar a struct TAluno contendo 4 atributos.

In [7]:
%%writefile cap7ex01.c
#include <stdio.h>
#include <string.h>

struct TAluno {
  char nome[50];
  int idade;
  char rua[50];
  int numero;
};

int main() {
  struct TAluno ana; // instancia uma varivel c do tipo TAluno
  // ENTRADA DE DADOS
  strcpy(ana.nome, "Ana Silva");
  ana.idade = 18;
  strcpy(ana.rua, "Avenida Paulista");
  ana.numero = 1000;

  // SADA DE DADOS
  printf("nome: %s\nidade: %d\n", ana.nome, ana.idade);
  printf("rua: %s\nnmero: %d\n", ana.rua, ana.numero);
  return 0;
}

Exemplo 05 - Criar dois registros¶

Exemplo para criar a struct Cliente contendo 3 atributos, sendo um deles uma outra estrutura Endereco.

In [61]:
%%writefile cap7ex05.c
#include <stdio.h>
#include <string.h>

typedef struct {
  char rua[50];
  int numero;
} Endereco;

typedef struct {
  char nome[50];
  int idade;
  Endereco end;
} Cliente;

void leiaCliente(Cliente* cliente) {
  fflush(stdin);
  printf("nome: ");
  fgets((*cliente).nome, 50, stdin);
  printf("idade: ");
  scanf("%d", &(*cliente).idade);
  fflush(stdin);
  printf("rua: ");
  fgets((*cliente).end.rua, 50, stdin);
  printf("numero: ");
  scanf("%d", &(*cliente).end.numero);
}

void escrevaCliente(Cliente cliente) {
  printf("nome: %s\nidade: %d\n", cliente.nome, cliente.idade);
  printf("rua: %s\nnumero: %d\n", cliente.end.rua, cliente.end.numero);
}

int main() {
  Cliente clientes[2];
  // ENTRADA DE DADOS
  for (int i = 0; i < 2; i++) {
    printf("Cadastrar dados do cliente %d\n", i + 1);
    leiaCliente(&clientes[i]);
  }

  // SADA DE DADOS
  for (int i = 0; i < 2; i++) {
    printf("Imprimir dados do cliente %d\n", i + 1);
    escrevaCliente(clientes[i]);
  }
  return 0;
}

Arquivos¶

Os arquivos armazenados em disco so considerados como uma sequncia de caracteres e podem ser acessados em memria atravs de um ponteiro para o primeiro caracter do arquivo (texto ou binrio), como segue:

FILE *fp;
fp = fopen("nomeArquivo.txt", "modo"); // modo  definido a seguir

Seguem as principais funes para manipular arquivos da biblioteca stdio.h:

Funo | Descrio :---------:|---------- fopen() | Abre arquivo fclose() | Fecha arquivo putc() | Escreve um caracter no arquivo fputc() | Igual putc() getc() | L um caracter do arquivo fgetc() | Igual getc() fseek() | Posiciona o arquivo em um determinado byte fprintf()| Semelhante ao printf(), mas para arquivo fscanf() | Semelhante ao scanf(), mas para arquivo feof() | Verifica final de arquivo ferror() | Verifica se ocorreu erro rewind() | Posiciona o ponteiro para o incio do arquivo remove() | Apaga arquivo fflush() | Descarrega o buffer do arquivo fgets() | Obtm uma string do arquivo fread() | L um bloco de dados do arquivo fwrite() | Escreve um bloco de dados no arquivo ftell() | Retorna a posio do ponteiro

| Modo | Arquivo | Funo | |:--------: |:-----------: |:---------------: | | r | Texto | Leitura | | w | Texto | Escrita em arquivo novo | | a | Texto | Escrita em arquivo existente | | r+ | Texto | Leitura/Escrita | | w+ | Texto | Leitura/Escrita | | a+ | Texto | Leitura/Escrita | | rb | Binrio | Leitura | | wb | Binrio | Escrita | | ab | Binrio | Escrita | | r+b | Binrio | Leitura/Escrita | | w+b | Binrio | Leitura/Escrita | | a+b | Binrio | Leitura/Escrita |

#include <stdio.h>

int main() {
  char* filename = "teste.txt";
  FILE* file;

  file = fopen(filename, "w"); // Cria arquivo para escrita
  if (file == NULL) {
    printf("Erro ao abrir o arquivo: %s\n", filename);
    return -1;
  }

  // SADA DE DADOS
  for (int i = 0; i < 10; i++)
    fprintf(file, "Linha %02d\n", i + 1); // Escrever algo no arquivo

  fclose(file); // fecha arquivo
  return 0;
}

Exemplo para ler um arquivo texto.

%%writefile cap7ex08.c
#include <stdio.h>

int main() {
  char* filename = "teste.txt", ch;
  FILE* file;

  // ENTRADA DE DADOS
  file = fopen(filename, "r"); // Cria arquivo para leitura
  if (file == NULL) {
    printf("Erro ao abrir o arquivo: %s\n", filename);
    return -1;
  }

  // SADA DE DADOS
  while ((ch = fgetc(file)) != EOF) { // L arquivo
    printf("%c", ch); // caracter por caracter
  }

  fclose(file); // fecha arquivo
  return 0;
}

int main() {
  Cliente clientes[20];
  char* filename = "dados.csv";

  // ENTRADA DE DADOS
  FILE* file; // Cria arquivo para leitura
  if ((file = fopen(filename, "r")) == NULL) {
    printf("Erro ao abrir o arquivo: %s\n", filename);
    return -1;

  int contador = 0;
  char linha[512]; // espao para cada linha lida
  while (fgets(linha, sizeof(linha), file))  // para cada linha
    criaCliente(&clientes[contador++], linha);

  for (int i = 0; i < contador; i++) {
    printf("\nImprimir dados do cliente %d\n", i + 1);
    escrevaCliente(clientes[i]);
  }

  fclose(file); // fecha arquivo
  return 0;
}

%%writefile myBiblioteca.c
#include "myBiblioteca.h"
// cria um cliente a partir de uma linha de texto. Exemplo:
// "Maria Souza, 20, Avenida Paulista, 1200"
void criaCliente(Cliente* cliente, char linha[512]) {
  // Ler nome
  char* token = strtok(linha, ","); // ler at a primeira ","
  strcpy((*cliente).nome, token); // nome

  // Ler idade
  token = strtok(NULL, ","); // continar lendo at a prxima ","
  (*cliente).idade = atoi(token); // converter str to int

  // Ler rua
  token = strtok(NULL, ",");
  strcpy((*cliente).end.rua, token + 1); // +1 retira 1o espao em branco

  // Ler numero
  token = strtok(NULL, ",");
  (*cliente).end.numero = atoi(token); // converter str to int
}
void escrevaCliente(Cliente cliente) {
  printf("nome: %s\nidade: %d\n", cliente.nome, cliente.idade);
  printf("rua: %s\nnmero: %d\n", cliente.end.rua, cliente.end.numero);
}

// leia um cliente a partir do teclado
// nome, idade, rua, numero
void leiaCliente(Cliente* cliente) {
  fflush(stdin);
  printf("nome: ");
  fgets((*cliente).nome, 50, stdin);
  printf("idade: ");
  scanf("%d", &(*cliente).idade);
  fflush(stdin);
  printf("rua: ");
  fgets((*cliente).end.rua, 50, stdin);
  printf("numero: ");
  scanf("%d", &(*cliente).end.numero);
}
