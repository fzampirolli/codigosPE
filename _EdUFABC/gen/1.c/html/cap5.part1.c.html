Revisão do capítulo anterior (Estruturas de Repetição - Laços)¶

  • Estruturas de repetio (laos) devem ser utilizadas quando existem instrues que se repentem e podem ser de vrios tipos, dependendo da linguagem utilizada, por exemplo:

    • do-while (no aceita em Python)
    • while (a maioria)
    • for (a maioria)
    • repeat (R)

  • O uso de laos depende da lgica a ser implementada e da linguagem utilizada. Mas, se tiver um nmero fixo de iteraes, geralmente se usa for.

  • Validao de dados utilizando laos:

    • Incluir um lao para verificar o valor lido.

  • Interrupo da execuo em laos:

    • Depende da linguagem, algumas possibilidades:
      • break - interrompe o lao
      • continue - no executa o final do lao
      • exit - aborta o lao e o programa!

  • Neste captulo iremos abordar a estrutura de Vetor, incluindo os conceitos apresentados nos captulos anteriores.

Introdução¶

  • Um vetor em programao formado por um conjunto de $n$ variveis de um mesmo tipo de dado (em geral), sendo $n$ obrigatoriamente maior que zero.
  • Essas variveis (ou elementos) so identificadas e acessadas por um nome e um ndice.
  • Na maioria das linguagens de programao, o ndice recebe valores de $0$ (primeiro elemento) at $n-1$ (ltimo elemento).
  • As variveis de um vetor so armazenadas em posies consecutivas de memria.

Trabalhando com vetores¶

  • Quando um vetor criado, instanciada uma varivel do tipo vetor (em algumas linguagens de programao, sendo necessrio se definir um nome, o tipo de dado e o tamanho do vetor).
  • Na linguagem C, por exemplo, a definio do tamanho de um vetor ocorre antes de compilar o programa.
    • Ou seja, o programador dever informar no cdigo a quantidade de memria a ser reservada para o vetor, atravs de um nmero inteiro positivo ou constante que o contenha, especificando o nmero de elementos do mesmo tipo que a memria reservada ir comportar.
    • Neste caso, uma varivel no pode ser usada.
  • J na linguagem Java, possvel alocar memria em tempo de execuo do programa.
    • Por exemplo, possvel criar um vetor para armazenar as notas de uma turma de alunos, onde o nmero de alunos uma varivel a ser lida durante a execuo do programa.

  • Em muitas linguagens, como Java e C, o processo de criar um vetor ocorre em dois passos distintos:

    1. primeiro se cria uma varivel de referncia para o vetor;
    2. em seguida se reserva a memria para um dado nmero de elementos do mesmo tipo.

  • No exemplo da figura abaixo,

    • a criao da varivel de referncia de um vetor v define apenas a posio de memria em hexadecimal 0A, onde ser armazenado o seu primeiro elemento.
      • neste exemplo, V[0] = -128 o primeiro elemento do vetor.
      • a quantidade de bytes por elemento vai depender to tipo de dado armazenado. Neste exemplo cada elemento ocupa um byte.
    • o segundo passo da criao de um vetor reservar (ou alocar) memria para todos os seus elementos (dependendo da linguagem).

    essa alocao de memria pode ocorrer em tempo de execuo, como ocorre em Python!

  • Note que a varivel v acima agora se refere a uma posio de memria e no a um elemento de dado.
  • Os elementos devem ser acessados atravs do ndice, um por vez.
  • Isso pode requerer uso de laos para leitura e impresso dos seus elementos, dependendo da linguagem de programao utilizada.
    • O ndice de vetor sempre um nmero inteiro.

Formas de Percorrer um Vetor¶

Percorrer um vetor com o laço para¶

  • Como um vetor, aps criado (alocado e com elementos), tem tamanho fixo (sempre com nmero de elementos n>0), geralmente indicado usar estruturas de repetio do tipo para (for), de forma a percorrer todos os elementos do vetor

    • assim tornando o cdigo genrico para um vetor de qualquer tamanho n.

  • Alm disso, natural percorrer (ou varrer) o vetor da posio i=0 at a posio i<n

    • O ndice i assume automaticamente os valores 0, 1, 2, at n-1, em cada iterao do lao.
    • Em algumas linguagens, como Matlab e R, o ndice assume valores entre 1 at n.
  • No exemplo a seguir em pseudocdigo, um vetor v criado com 6 posies e o lao para inicializa todos os seus elementos com o valor 0.
  • A forma natural de percorrer os elementos de um vertou (ou varrendo o vetor) na ordem raster, do primeiro at o ltimo elemento.
  • A vantagem de se usar a estrutura para para varrer um vertor permitir embutir na prpria sintaxe (linha) da instruo:
    • declarao e inicializao do ndice,
    • incremento e
    • condio de sada do lao.

Pseudocódigo¶

Instanciar um vetor v de inteiro com 6 elementos (n=6)
para cada indice i, de i=0; at i<n; passo i=i+1 faa
    v[i] = 0

Percorrer um vetor com o laço enquanto¶

  • Se o programador preferir, ou o problema a ser resolvido exigir, o vetor pode alternativamente ser percorrido utilizando-se uma estrutura de repetio do tipo enquanto:

Pseudocódigo¶

n=6
vetor v de inteiros com n elementos 
inteiro i=0
enquanto i<n faa {
    v[i] = 0
    i=i+1
}
  • O cdigo usando a estrutura de repetio enquanto produz o mesmo resultado que usando com para, porm, usando mais instrues:
    • a criao do contador i
    • o incremento i=i+1.
  • Essas duas instrues ficam encapsuladas na prpria instruo para.

Outras formas de percorrer um vetor¶

  • Dependendo do problema a ser tratado, existem vrias formas de se varrer um vetor usando estruturas de repetio para acessar cada elemento.
  • Por exemplo, possvel percorrer o vetor do ltimo elemento para o primeiro (essa varradora chamada de anti-raster):

Pseudocódigo: percorrer um vetor usando para na ordem inversa (anti-raster).¶

Instanciar um vetor v de inteiro com 6 elementos (n=6)
para cada indice i, de i=n-1; at i<=0; passo i=i-1 faa
    v[i] = 0
  • Tambm, possvel percorrer somente alguns elementos do vetor, como para acessar os elementos que esto nos ndices pares ou mpares, diferenciadamente:

Pseudocódigo: percorrer um vetor usando para, com passo 2.¶

vetor v de inteiro com 6 elementos
n=6
para cada indice i, de i=0; at i<n-1; passo i=i+2 faa {
    v[i] = 0
    v[i+1] = 1
}

Modularização e Vetores¶

  • Como uma boa prtica de programao, alm de comentar os cdigos e organizar com tabulao, como apresentado nos exemplos deste livro,

    • tambm recomendvel modularizar o cdigo usando mtodos, como apresentado no Captulo 2.

  • Todo sistema computadorizado de informao possui trs partes bem definidas, agrupadas em

    • mdulo(s) de entrada,
    • mdulo(s) de processamento e
    • mdulo(s) de sada.

  • Ao manipular informaes armazenadas em vetores, natural criar tambm pelo menos trs mdulos ou mtodos:

    • leiaVetor,
    • processaVetor e
    • escrevaVetor,

    satisfazendo essa definio de sistema de informao.

Para a reutilizao de cdigo, os mdulos leiaVetor e escrevaVetor podero ser muito reutilizados para resolver outros problemas de manipulao de vetores.

mtodo escrevaVetor(inteiro v[], inteiro n):
    para cada ndice i, de i=0; at i<n; passo i=i+1 faa

  }
  return v;

int main(void) {

  // ENTRADA DE DADOS
  int n, *ras, *notas;   // variaveis de referncia ras e notas

  int contador = 0;

    }
  }

  free(notas);
  return 0;
}

Exemplo 04 - Aplicação "dilata" vetor¶

  • O programador deve tomar cuidado ao acessar os elementos de um vetor para no ultrapassar os seus limites alocados previamente, ou seja,

    • para um vetor v de tamanho n, ndices i com i<0 ou i>=n no existem.

  • O exemplo a seguir ilustra este problema.

  • Considere o problema de "dilatar" um vetor.

    • O objetivo criar um vetor v1 de inteiros com n posies e um outro vetor v2 de inteiros tambm com n posies.
    • Cada posio i de v2 armazena o clculo do mximo entre cada elemento i em v1 e seus vizinhos:
      1. esquerda v1[i-1],
      2. do prprio elemento v1[i] e
      3. do seu vizinho direita v1[i+1].

  • Veja uma ilustrao da operao de dilatao na Figura abaixo, onde v1 o vetor de entrada e v2 o vetor de sada, contendo a "dilatao" de v1, seguido do cdigo para resolver este problema proposto.

Pseudocódigo¶
mtodo escrevaVetor(inteiro v[], inteiro n):
    para cada ndice i, de i=0; at i<n; passo i=i+1 faa
        escreva(" " + v[i]);

mtodo leiaVetor(inteiro n): retorna vetor de inteiro v[]
    vetor v de inteiros com n elementos
    para cada ndice i, de i=0; at i<n; passo i=i+1 faa
        v[i] = leia("Entre com o elemento " + i + ":");
    retorne v

// Inicializaes
inteiro max=0
// ENTRADA
inteiro n = leia("Digite o tamanho do vetor:");
vetores v1 e v2 de inteiros com n elementos

inteiro v1[] = leiaVetor(n)

// PROCESSAMENTO
para cada ndice i, de i=0; at i<n; passo i=i+1 faa
    max = v[i]
    se i-1 >= 0 e max < v1[i-1] faa
        max = v1[i-1]
    se i+1 < n e max < v1[i+1] faa
        max = v1[i+1]
    v2[i] = max

// SADA
escrevaVetor(v2)

  v1 = leiaVetor(n);
  // PROCESSAMENTO
  for (int i = 0; i < n; i++) {
    int max = v1[i];
    if (i-1 >= 0 && max < v1[i-1])
        max = v1[i-1];
    if (i+1 < n && max < v1[i+1])
        max = v1[i+1];
    v2[i] = max;
  }
  // SADA:
  printf("\nv2:\n");
  escrevaVetor(v2,n);
  free(v1); // liberar memria alocado com malloc
  free(v2);
  return 0;
}

%%writefile str2int.c
#include <stdio.h>
#include <string.h>
#include <stdlib.h>

int main() {
   char string[] = "7 3 7 9 7 7 0 9 8 4 8 9 0 1     7 8 4 1 1 0     ";
   char * token = strtok(string, " ");
   int v[100];
   int i = 0;
   while( token != NULL ) {
      v[i++] = atoi(token); // converte texto para inteiro
      token = strtok(NULL, " ");
   }
   int n=i;
   for (i=0;i<n;i++ ){
     printf("%d ",v[i]);
   }
   return 0;
}
